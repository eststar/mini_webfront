# **PEECE MAKER 웹 프론트엔드 개발 결과 보고서**

## **1. 프로젝트 개요**
*   **프로젝트명**: PEECE MAKER (피스메이커)
*   **서비스 목적**: 제주도 내 공중화장실의 위치 및 상세 정보(편의시설, 안전시설 등)를 제공하고, 사용자 리뷰 및 커뮤니티 기능을 통해 쾌적한 화장실 이용을 돕는 웹 서비스입니다.
*   **개발 환경**: Windows (win32)
*   **주요 기술 스택**:
    *   **Core**: Next.js 16 (App Router), React 19, TypeScript
    *   **Styling**: Tailwind CSS 4, Framer Motion (애니메이션), Next-themes (다크모드 지원)
    *   **Visualization**: Recharts (차트), React-Kakao-Maps-SDK (지도)
    *   **Networking**: Fetch API (백엔드 연동)

---

## **2. 주요 기능 및 구현 내용**

### **A. 랜딩 페이지 (`src/app/page.tsx`)**
*   **인트로 디자인**: 제주도 배경 영상(`jeju.mp4`)과 스크롤에 따른 텍스트 애니메이션(Framer Motion)을 적용하여 몰입감 있는 첫인상을 제공합니다.
*   **정보 소개**: 프로젝트의 핵심 기능(Map, Chart, Board), 데이터 출처(제주시 공공데이터), 개발팀 소개 섹션으로 구성되어 있습니다.

### **B. 화장실 지도 (`src/app/main/mapView.tsx`)**
*   **Kakao Map 연동**: 카카오맵 API를 활용하여 사용자 위치 기반으로 주변 화장실을 마커로 표시합니다.
*   **클러스터링**: 다수의 화장실 마커가 겹칠 경우 클러스터링(묶음) 처리하여 지도 가시성을 확보했습니다.
*   **상세 필터링**:
    *   **시설 필터**: 안심시설(비상벨/CCTV), 장애인용, 유아동반 시설 여부로 필터링 가능.
    *   **성별 필터**: 남성용/여성용 화장실 보유 여부에 따른 필터링 제공.
*   **상세 정보 팝업**: 마커 클릭 시 해당 화장실의 세부 정보 및 리뷰를 확인할 수 있는 팝업(`ToiletPopup`)을 띄웁니다.

### **C. 통계 대시보드 (`src/app/main/chartView.tsx`)**
*   **데이터 시각화 (Recharts 활용)**:
    *   **지역별 분포 (BarChart)**: 읍/면/동 단위로 화장실 개수를 집계하여 막대 그래프로 시각화하였습니다.
    *   **남녀 수용력 (PieChart)**: 전체 또는 필터링된 화장실의 남성(대/소변기) vs 여성(대변기) 수용 시설 비율을 도넛 차트로 제공합니다.
*   **동적 필터링**:
    *   상단 버튼(전체, 안심, 장애인, 유아) 클릭 시 `useMemo`를 통해 즉시 통계 데이터를 재계산하여 차트와 랭킹에 반영합니다.
    *   다크 모드에 대응하여 차트의 축 색상(`tickColor`) 등이 자동으로 변경됩니다.
*   **랭킹 시스템**:
    *   **Best/Worst 5**: 리뷰 평점 데이터를 기반으로 상위 5곳과 하위 5곳의 화장실을 리스트업합니다.
    *   필터링된 화장실 목록 내에서만 랭킹을 다시 산출하여, 특정 조건(예: 장애인 화장실) 내에서의 평점 순위를 확인할 수 있습니다.

### **D. 커뮤니티 게시판 (`src/app/main/boardView.tsx`)**
*   **CRUD 구현**: 게시글 목록 조회, 작성, 상세 보기, 수정, 삭제 기능을 백엔드 API와 연동하여 구현했습니다.
*   **UI/UX**:
    *   **검색 기능**: 제목, 내용, 작성자 닉네임으로 게시글을 실시간 필터링(검색) 할 수 있습니다.
    *   **페이지네이션**: 게시글이 많을 경우 페이지를 나누어 보여주는 로직을 포함하고 있습니다.
    *   **모션 효과**: 게시글 목록 진입, 상세 보기 전환 시 부드러운 전환 애니메이션을 적용했습니다.

## **3. 폴더 구조 (핵심)**
```text
src/
├── app/
│   ├── main/             # 메인 기능 모듈
│   │   ├── chartView.tsx # 통계 차트 컴포넌트
│   │   ├── mapView.tsx   # 지도 컴포넌트
│   │   ├── boardView.tsx # 게시판 컴포넌트
│   │   └── page.tsx      # 메인 페이지 진입점
│   ├── page.tsx          # 랜딩 페이지
│   └── globals.css       # 전역 스타일 (Tailwind 설정)
├── components/           # 공통 컴포넌트 (테마 프로바이더 등)
└── assets/               # 더미 데이터 및 정적 리소스
```

## **4. 종합 의견**
본 프로젝트는 **Next.js 16**과 **React 19** 최신 버전을 기반으로 구축되어 성능과 유지보수성이 뛰어납니다. 특히 **Tailwind CSS**와 **Framer Motion**을 적극 활용하여 시각적으로 세련되고 인터랙티브한 사용자 경험(UX)을 제공하고 있습니다. 기능적으로는 단순 정보 제공을 넘어, 공공데이터와 사용자 리뷰 데이터를 결합하여 **지도-통계-커뮤니티**로 이어지는 유기적인 서비스를 성공적으로 구현한 것으로 판단됩니다.
